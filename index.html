Components in React are the building blocks of the user interface.

Properties (props) and state
    Properties (props)
  • data that are passed to a component from the outside
  • read-only
  • used to pass information from parent components to child components
  • are passed to a component as attributes in JSX
    State
  • internal data of the component that can change during the component's lifecycle
  • changing the state triggers a re-render of the component, allowing the user interface to be updated when the data changes
Component Lifecycle
The Component Lifecycle in React refers to the series of events that occur during the lifespan of a component. Such as initialization, update and remove.
    Some of the most commonly used lifecycle methods include:
  • componentDidMount()
  • componentDidUpdate()
  • componentWillUnmount()
Understanding when lifecycle methods are called and how to use them allows for effectively managing the behavior of components in various situations.
How to use the components effectively?
Build flexible component. Reuse it!
Separate components by functionality or by individual parts of the user interface.
Avoid state changes directly.
